Build a simple web app called Creative Performance Analyzer.

Purpose:
This app helps a paid social growth team upload creative performance data and quickly understand which ads are winning, which are inefficient, and what to test next.

Use case:
The user uploads a CSV or Excel file with ad creative data. The app cleans the data, calculates key KPIs, ranks creatives by the selected campaign goal, and generates simple recommendations.

Tech stack:
Use Python with Streamlit.
Use pandas for data processing.
Use plotly or matplotlib for charts.
The app should run fully inside Replit.

Core features:

1. File upload
Allow the user to upload CSV or Excel files.
Support columns like:
Creative ID
Platform
Format / Concept
Length
Spend
Thumbstop Rate
6s Hold Rate
CTR
Trial Starts
Paid Starts

The app should handle slightly different column names by normalizing them.
For example:
"Paid Starts", "paid_starts", "Paid starts" should be treated the same.
"Spend" may include dollar signs or commas and should be converted to numeric.
Percent columns like CTR, Thumbstop Rate, and 6s Hold Rate should be converted into decimals or clean numeric percentages.

2. KPI calculations
After upload, calculate these KPIs when the required columns exist:

Cost per paid start = Spend / Paid Starts
Cost per trial start = Spend / Trial Starts
Trial to paid conversion rate = Paid Starts / Trial Starts
Paid starts per $1,000 spend = Paid Starts / Spend * 1000
Trial starts per $1,000 spend = Trial Starts / Spend * 1000
CTR efficiency score = CTR / Cost per paid start
Thumbstop efficiency score = Thumbstop Rate / Cost per paid start
Hold efficiency score = 6s Hold Rate / Cost per paid start

Avoid divide by zero errors.
If a KPI cannot be calculated because a column is missing, skip it and show a small warning.

3. Goal selector
Add a dropdown called "Campaign Goal".
Options:
Paid Starts
Trial Starts
Efficient Paid Starts
Efficient Trial Starts
Creative Engagement
Full Funnel Quality

Ranking logic:
Paid Starts:
Rank by Paid Starts descending.

Trial Starts:
Rank by Trial Starts descending.

Efficient Paid Starts:
Primary metric is Cost per Paid Start ascending.
Secondary metric is Paid Starts descending.
Target CPA is $90.
Flag creatives under $90 as "Scale Candidate".

Efficient Trial Starts:
Primary metric is Cost per Trial Start ascending.
Secondary metric is Trial Starts descending.

Creative Engagement:
Use a weighted score:
40% Thumbstop Rate
30% 6s Hold Rate
30% CTR

Full Funnel Quality:
Use a weighted score:
30% Cost per Paid Start
25% Paid Starts
20% Trial to Paid Conversion Rate
15% CTR
10% Thumbstop Rate

For cost per paid start, lower is better.
For all other metrics, higher is better.
Normalize metrics before combining them into a score.

8. Test recommendations
Add a final section called "What should we test next?"
Generate rule-based recommendations such as:
If TikTok has higher thumbstop but weaker paid conversion, recommend testing stronger product proof or clearer CTA.
If Meta has lower thumbstop but stronger paid conversion, recommend testing more scroll-stopping hooks while keeping the same offer.
If a creative has strong CTR but high CPA, recommend testing landing page or trial-to-paid flow.
If a creative has high thumbstop and low CTR, recommend testing a clearer value prop in the first 3 seconds.
If one concept has low CPA, recommend remixing that concept across platforms and lengths.

Do not use an LLM API. Keep recommendations rule-based.
